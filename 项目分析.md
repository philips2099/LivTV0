## 核心组件分析

### 1. MainActivity

**职责**：应用主入口，管理Fragment生命周期、用户交互、手势识别

**关键功能**：

| 功能 | 实现方式 |
|------|----------|
| 全屏沉浸模式 | WindowInsetsController + SYSTEM_UI_FLAG |
| 手势操作 | GestureDetector (单击/双击/长按/滑动) |
| 频道切换 | 上下滑动或按键切换 |
| 音量/亮度调节 | 右侧滑动调音量，左侧滑动调亮度 |
| 按键事件处理 | onKeyDown 处理遥控器按键 |

**手势交互设计**：
- **单击**：显示频道菜单
- **双击**：显示设置界面
- **长按**：显示节目指南(EPG)
- **上下滑动(中间区域)**：切换频道
- **左侧滑动**：调节亮度
- **右侧滑动**：调节音量

---

### 2. MainViewModel

**职责**：数据管理中心，处理频道列表、EPG数据、配置加载

**核心流程**：

```
初始化 → 加载缓存频道 → 解析频道数据 → 构建TVGroupModel → 更新EPG → 通知UI
```

**频道格式支持**：
- JSON数组格式 `[{...}, {...}]`
- M3U格式 `#EXTM3U ...`
- 简易格式 `频道名,url1,url2,...`

**EPG更新机制**：
- 支持从URL加载XML格式EPG
- 缓存到本地文件 `epg.xml`
- 通过频道名称模糊匹配EPG数据

---

### 3. PlayerFragment

**职责**：视频播放核心组件，基于ExoPlayer

**播放策略**：

| 场景 | 处理方式 |
|------|----------|
| 播放错误 | 自动重试，切换源类型(HLS/PROGRESSIVE) |
| 多URL支持 | 依次尝试多个备用地址 |
| 软解码切换 | 根据SP.softDecode配置切换渲染器模式 |

**SourceType自动检测**：
- `.m3u8` → HLS
- `.mpd` → DASH
- `rtsp://` → RTSP
- `rtmp://` → RTMP
- 其他 → HLS + PROGRESSIVE

## 代码优化建议

### 1. 代码重复问题

**问题**：`prev()` 和 `next()` 方法逻辑高度相似

**优化方案**：提取公共方法

```kotlin
// MainActivity.kt
private fun switchChannel(next: Boolean, keepInLikeMode: Boolean = false): Boolean {
    val prevGroup = viewModel.groupModel.positionValue
    val tvModel = if (SP.defaultLike && viewModel.groupModel.isInLikeMode && viewModel.groupModel.getFavoritesList() != null) {
        if (next) viewModel.groupModel.getNext(true) else viewModel.groupModel.getPrev(true)
    } else {
        if (next) viewModel.groupModel.getNext() else viewModel.groupModel.getPrev()
    }
    
    tvModel?.setReady()
    viewModel.groupModel.setPositionPlaying()
    viewModel.groupModel.getCurrentList()?.setPositionPlaying()
    
    val currentGroup = viewModel.groupModel.positionValue
    if (currentGroup != prevGroup) {
        menuFragment.updateList(currentGroup)
    }
    return tvModel != null
}

fun prev() = switchChannel(next = false)
fun next() = switchChannel(next = true)
```

### 3. Fragment管理改进

**问题**：Fragment实例直接作为成员变量，可能导致内存泄漏

**优化方案**：使用 `FragmentManager.findFragmentByTag`

```kotlin
private fun getPlayerFragment(): PlayerFragment {
    return supportFragmentManager.findFragmentByTag("player") as? PlayerFragment 
        ?: PlayerFragment().also {
            supportFragmentManager.beginTransaction()
                .add(R.id.main_browse_fragment, it, "player")
                .commit()
        }
}
